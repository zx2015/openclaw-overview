# OpenClaw CLI 生命周期与执行逻辑

OpenClaw 的 CLI 启动流程经过高度优化，确保了响应速度与用户体验。

## 1. 总入口：`src/entry.ts`
`src/entry.ts` 是整个 CLI 程序的门面，负责预处理环境并确保启动的平滑性：
1.  **参数标准化**：处理不同操作系统（特别是 Windows）下的参数兼容性，并初始化环境变量。
2.  **快速响应路径**：对 `--version` 和 `--help` 进行了逻辑前置优化，跳过复杂的模块加载，实现毫秒级反馈。
3.  **进程重启动 (Respawn)**：
    *   通过重启进程并注入 `--disable-warning=ExperimentalWarning`，自动隐藏 Node.js 的实验性特性警告。
    *   这一步确保了终端输出的整洁，避免用户被非关键的警告信息干扰。
4.  **核心移交**：在环境准备就绪后，动态导入并调用 `src/cli/run-main.js` 中的 `runCli` 函数。

## 2. 执行起点：`src/cli/run-main.ts`
`src/cli/run-main.ts` 是命令处理的真正核心，负责编排具体的指令逻辑：
1.  **配置载入**：自动读取并加载当前目录或指定位置的 `.env` 配置文件。
2.  **动态注册机制**：
    *   **核心命令**：注册基础的启动、配置和诊断命令。
    *   **子 CLI (Sub-CLIs)**：加载特定领域的子命令集（如 `skills` 管理）。
    *   **插件 CLI**：扫描并注册已安装插件通过 Plugin SDK 暴露的自定义命令。
3.  **异步解析与分发**：
    *   基于 `Commander` 构建命令程序。
    *   执行异步参数解析，并根据匹配结果调用对应的 Action 函数。
