# OpenClaw 记忆管理机制

OpenClaw 采用了一个多层级、可持久化且具备自我维护能力的记忆系统。它不依赖于复杂的单一数据库，而是巧妙地结合了 **人类可读的 Markdown 文件**、**向量搜索 (RAG)** 以及 **动态上下文压缩** 技术。

## 1. 记忆的四个维度

### 短期记忆：会话历史与压缩 (Session History & Compaction)
这是 AI 正在进行的对话上下文，用于维持当前任务的连贯性。
*   **存储方式**：内存中的消息队列。
*   **压缩机制 (Compaction)**：
    *   **触发条件**：当对话长度接近模型上下文窗口 (Context Window) 限制时。
    *   **核心逻辑**：系统启动一个专项“压缩任务”，让 AI 对既往对话进行摘要 (Summarization)，提取核心事实、决策和待办事项。
    *   **效果**：旧的详细对话被精炼摘要替换，在有限的 Token 消耗下实现超长对话的逻辑一致性。

### 长期记忆：工作区准则与事实 (Workspace Instructions & Facts)
定义了 AI 的“价值观”、“操作习惯”以及“核心事实库”。
*   **文件构成**：
    *   `AGENTS.md` (操作逻辑)。
    *   **[`MEMORY.md`](../docs/reference/templates/memory_root.md)** (长期事实与智慧库)：存储项目背景、重要决策及提炼后的客观事实。
    *   `SOUL.md` (性格边界)。
    *   `USER.md` (用户偏好)。
*   **特点**：在每次会话开始时都会完整（或截断后）注入 System Prompt，确保 AI 始终“守规矩”。

### 情节记忆：每日日志 (Daily Episodic Memory)
记录过去发生过的具体事件与任务细节。
*   **存储位置**：工作区下的 `memory/` 文件夹。
*   **存储格式**：按日期命名的 Markdown 文件（如 `memory/2026-03-10.md`）。
*   **记录内容**：Agent 主动记录当天的重大事项、遇到的挑战及解决方案。
*   **检索逻辑**：系统自动加载“今天”与“昨天”的日志；更久远的记录则通过语义搜索获取。

### 语义检索机制 (RAG - Retrieval-Augmented Generation)
当涉及历史跨度较大的提问时，OpenClaw 启用语义搜索：
*   **技术栈**：底层集成 `sqlite-vec` 提供向量存储与相似度匹配。
*   **工具支持**：
    *   `memory_search`：发起语义查询，在全量记忆文件中寻找匹配片段。
    *   `memory_get`：精准读取搜索到的特定行或段落。
*   **引用模式 (Citations)**：AI 在回答时会标注来源（如 `Source: memory/2025-12-01.md#L42`），方便用户溯源。

## 2. 核心机制：自我进化 (Self-Maintenance)

OpenClaw 的记忆系统是“动态演进”的，AI 拥有对自己记忆的编辑权限：
1.  **主动记录 (Active Logging)**：任务结束或有重大发现时，Agent 自主调用写入工具记录至每日日志。
2.  **准则更新 (Rule Updating)**：若发现某条既有规则已过时，Agent 会主动修改 `AGENTS.md` 以优化未来的行为。
3.  **冲突解决**：当新旧经验冲突时，Agent 负责更新结论并在交互中告知用户逻辑的演变。

## 3. 记忆分层对照表

| 记忆层级 | 存储位置 | 刷新频率 | AI 的视角 |
| :--- | :--- | :--- | :--- |
| **瞬时记忆** | 聊天历史 (RAM) | 每一轮对话 | “我们刚刚聊了什么。” |
| **短期总结** | Compaction 摘要 | 接近 Token 限制时 | “谈话的核心事实是什么。” |
| **情节记忆** | `memory/YYYY-MM-DD.md` | 任务结束/每天 | “过去某天我做了什么。” |
| **长期记忆** | `AGENTS.md`, `SOUL.md` | 重大变更/用户指令 | “我是谁，我该怎么干活。” |

## 4. 设计哲学：透明化与可控性

OpenClaw 避开了黑盒索引，转而使用 **Markdown** 作为核心存储介质。用户可以随时打开 `~/.openclaw/workspace` 文件夹，像看日记一样查看、修改或删除 AI 的记忆。这种设计确保了 AI 的行为演化轨迹对用户是完全透明且可干预的。

关于在多代理环境下的物理隔离、逻辑共享及记忆权属关系，请参阅 **[记忆归属与层级架构](./memory_hierarchy.md)**。
