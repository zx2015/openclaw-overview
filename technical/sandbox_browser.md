# OpenClaw 沙箱：浏览器专项隔离

OpenClaw 针对 `browser` 工具（基于 Playwright）实现了物理级的专项隔离与可视化桥接。

## 1. 独立容器策略
不同于常规工具，浏览器默认运行在专门的容器中（通常基于 `Dockerfile.sandbox-browser`）：
*   **资源密集**：浏览器需要更多的内存和图形渲染库。
*   **安全屏障**：防止恶意网站通过浏览器引擎漏洞实现容器逃逸攻击宿主机。

## 2. 桥接架构：Browser Bridge Server
由于浏览器运行在容器内，OpenClaw 通过以下方式实现控制：
1.  **端口映射**：将容器内的 CDP (Chrome DevTools Protocol) 端口映射至宿主机的回环地址（`127.0.0.1`）。
2.  **代理服务**：启动一个 Browser Bridge Server，作为 Gateway 与沙箱浏览器之间的中间层，负责指令转发与安全鉴权。

## 3. 可视化监控与 noVNC
为了让用户能实时观察 AI 的网页操作，系统集成了可视化能力：
*   **动态授权**：系统实时生成动态密码。
*   **Token 化访问**：通过 WebSocket 提供 Token 化的 noVNC URL。
*   **实时画面**：用户可以通过 OpenClaw Web 界面直接看到容器内浏览器的渲染画面。

## 4. 隔离安全性
*   **无状态浏览**：默认情况下，每次浏览器会话都可以配置为无状态模式，关闭即销毁所有 Cookie 和缓存。
*   **网络收敛**：浏览器沙箱可配置独立的安全组规则，限制其只能访问必要的公网地址。
