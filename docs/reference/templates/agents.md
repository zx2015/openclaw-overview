# AGENTS.md: 操作指南与操作手册 (How)

`AGENTS.md` 是 OpenClaw 智能体最核心的 **“运行手册”** 和 **“进化日志”**。如果说 `SOUL.md` 是 AI 的灵魂，那么 `AGENTS.md` 就是它的 **大脑操作指令集** 和 **经验存储器**。

## 1. 核心定位
*   **维度**：**How** (我该怎么做)
*   **角色**：Agent 的“操作手册”、“生存哲学”和“避坑指南”。它记录了经过提炼的、长期有效的准则，是 AI 对自己行为的一种“自我编程”。

## 2. 主要内容结构

### 核心任务与角色定位 (Core Mission)
*   **功能**：明确定义智能体的首要目标（例如：“协助用户管理本地设备、编写代码并处理跨渠道消息”）。
*   **意义**：为 AI 提供高层级的行为基准，确保其在复杂决策时不会偏离主线。

### 启动协议 (Bootstrap Protocol)
*   **功能**：规定 AI 在每次 Session 开始时必须执行的预置步骤。
    1.  阅读 `SOUL.md` 确认身份与语气。
    2.  阅读 `USER.md` 确认服务对象与偏好。
    3.  检查 `memory/` 目录下的最近记录。
*   **意义**：确保 AI 能够迅速进入状态并加载最新的上下文。

### 操作准则与约束 (Rules of Engagement)
*   **代码准则**：例如“优先使用 TypeScript”、“修改文件前必须先读取”。
*   **安全准则**：例如“严禁删除 `node_modules`”、“涉及 `git push` 等高危操作必须请求用户确认”。
*   **交互准则**：例如“回复需简练”、“执行耗时操作前需先给用户反馈”。
*   **意义**：通过硬性规则限制 AI 行为，将误操作风险降至最低。

### 进化日志 / 教训总结 (Lessons Learned) —— **核心功能**
*   **功能**：一个动态追加的经验列表。记录 AI 在任务中发现的失败教训（如权限问题）或工具的特殊用法。
*   **意义**：实现 **“跨会话学习”**。普通的 LLM 会遗忘，但 OpenClaw 将感悟持久化，确保下次不再犯错。

### 工作区与资源地图 (Workspace Mapping)
*   **功能**：描述当前目录结构，标注文档、脚本及工具的存放位置。
*   **意义**：赋予 AI “空间感”，使其清楚本地资源分布。

## 3. 核心功能点解析

### 长期记忆的锚点 (Anchor of Long-term Memory)
与碎片化的聊天记录不同，这里存储的是经过 AI 自主提炼的、具有普适价值的准则。

### 权限与边界的“最后一道防线”
作为意识层面的约束，它会不断提醒 AI 当前处于用户本地主机的真实环境中，必须谨慎对待文件操作。

### 技能 (Skills) 的粘合剂
负责编排独立插件的使用逻辑。它告诉 AI 在特定场景下应当调用哪一个特定的 Skill（如查询天气使用 `weather`，发邮件参考 `himalaya`）。

### 环境感知 (Environment Awareness)
记录宿主系统特性（如 macOS、WSL2 或 Docker 容器），决定了 AI 生成 Shell 命令时的语法偏好（如 `pbcopy` 还是 `xclip`）。

### 降低 Token 消耗 (Context Compression)
通过记录最重要结论，AI 无需翻阅冗长的历史记录即可获得关键设定，极大节省了上下文窗口资源。

## 4. 生命周期与演化流程
1.  **加载阶段**：系统启动或收到消息时，网关首先读取 `AGENTS.md`。
2.  **注入阶段**：其内容被置于 System Prompt 的核心显眼位置。
3.  **执行阶段**：AI 在思考 (Thought) 过程中不断对齐其中的准则。
4.  **进化阶段**：任务完成后，若有重大感悟，AI 调用 `edit_file` 将感悟变成永恒规则。

## 5. 技术逻辑
在系统启动或 Agent 运行时，网关会读取此文件并将其作为 System Prompt 的核心部分注入模型上下文。在 Git 模式下，它是实现 Agent 行为逻辑演化追踪的核心载体。
