# AGENTS.md: 操作指南与操作手册 (How)

`AGENTS.md` 是 OpenClaw 智能体最核心的 **“运行手册”** 和 **“进化日志”**。如果把 OpenClaw 比作一家公司，`AGENTS.md` 就是那本集 **《员工手册 + 岗位操作规程 + 避坑指南》** 于一体的百科全书。

## 1. 核心定位
*   **维度**：**How** (我该怎么做)
*   **角色**：Agent 的大脑操作指令集与长期记忆锚点。它负责将通用的 LLM 约束为一个“懂规矩、会干活、能吸取教训”的专业本地代理。

## 2. 行为准则与交互协议 (Behavioral Protocol)
规定了 AI 在“开口说话”前必须遵守的职场纪律：
*   **简洁性原则**：避免废话，拒绝冗长的开场白，直接提供核心答案。
*   **工具调用叙述 (Tool Narration)**：
    *   **低风险操作**：如读取文件、列目录，不解释，直接执行。
    *   **敏感/高风险操作**：如删除、执行复杂脚本，必须在执行前进行简短的意图说明。
*   **静默回复 (Silent Replies)**：规定在无需用户介入的后台处理阶段，使用 `SILENT_REPLY_TOKEN` 保持静默。
*   **审批引导**：当 `exec` 返回“等待审批”状态时，AI 必须完整展示命令并引导用户通过 `/approve` 进行授权。

## 3. 工具调用的最佳实践 (Tooling Logic & Strategy)
教导 AI 如何“聪明地”使用工具，而非机械调用：
*   **文件操作规范**：坚持“先读后写”；优先使用 `apply_patch` 进行精准的行级修改，避免大规模全量覆盖导致的代码污染。
*   **长时任务管理**：严禁死循环轮询 (Polling)，学会使用 `yieldMs` 或启动独立的后台进程。
*   **任务拆分 (Sub-agents)**：明确定义了“分身”时机。当任务复杂度过高、耗时极长或需在干净沙箱内运行（隔离风险）时，必须通过 `sessions_spawn` 启动子代理。

## 4. 技能 (Skills) 的发现与调度
作为系统的“总调度员”，编排各类独立插件：
*   **自动扫描**：要求 AI 在处理特定垂直领域任务前，先扫描 `<available_skills>` 列表。
*   **深度学习**：识别到匹配技能（如 `mcporter` 或 `github`）后，AI 必须先读取该技能对应的 `SKILL.md`，彻底理解参数规范后再行调用。
*   **冲突处理**：规定当内置工具与外部技能功能重叠时，根据当前上下文优先级选择最优执行路径。

## 5. 记忆的固化与知识整理 (Knowledge Consolidation) —— **进化的核心**
这是实现 AI “自我演化”的关键机制：
*   **教训记录 (Lessons Learned)**：动态追加 AI 在实战中发现的失败尝试（如：“在 WSL2 环境下不能直接调用 `open` 命令”）。
*   **记忆路由逻辑**：
    *   **`MEMORY.md`**：进长期客观事实。
    *   **`memory/`**：进每日流水日志。
    *   **`AGENTS.md`**：进规则改变与行为修正。
*   **自我修正**：若用户纠正了 AI 的某个行为，AI 必须立即将此纠正转化为新规则写入此文件。

## 6. 运行环境的“本地化”适配 (Context-Awareness)
赋予 AI “空间感”与“安全意识”：
*   **环境感知**：明确告知 AI 当前运行在原生 macOS、Docker 沙箱还是远程 Linux。
*   **路径映射**：管理 `displayWorkspaceDir`，并处理宿主机路径与容器隔离路径的映射关系。
*   **权限边界**：严禁修改 `node_modules` 或未经授权访问私密文件夹（如 Photos/Documents）。

## 7. 总结：AGENTS.md 的决策工作流
每当接收到新消息，AI 均按以下逻辑闭环思考：
1.  **[对齐身份]**：我的语气是否遵循了 `SOUL.md`？
2.  **[环境检查]**：我在哪个路径？当前是什么操作系统？
3.  **[寻找工具]**：内置工具有没有？有没有专门的 `SKILL.md` 可用？
4.  **[决策执行]**：是直接回复，还是调工具，还是派个“分身”子代理？
5.  **[复盘总结]**：这事办得漂亮吗？有没有新坑需要记在末尾的“教训”里？

## 8. 技术实现与安全隔离
*   **注入机制**：在系统启动或 Agent 运行时，网关在宿主机读取此文件并将其作为 System Prompt 的核心部分注入。
*   **沙箱隔离**：出于安全考虑，`AGENTS.md` **默认不会**被挂载或拷贝到沙箱环境中。这确保了沙箱内的代码无法通过文件系统访问或修改 AI 的核心指令集，实现了“大脑认知”与“物理执行”的物理级解耦。
*   **版本追踪**：在 Git 模式下，它是实现 Agent 行为逻辑演化追踪的核心载体。
