# USER.md: 用户偏好与背景 (For Whom)

`USER.md` 是 OpenClaw 智能体工作区中对“服务对象”的详细记录。

## 1. 核心定位
*   **维度**：**For Whom** (用户是谁)
*   **角色**：Agent 的服务对象肖像与用户习惯。

## 2. 具体用途
*   **用户背景肖像**：记录用户的姓名、职业、地理位置以及联系信息。
*   **偏好设定**：包含用户对回复详细程度、语言习惯、默认时间格式及地理位置信息的偏好。
*   **个性化关联**：作为 AI 提供定制化回复（如本地天气、特定工作建议）的重要参考背景。
*   **引导录入**：在入职流程中，系统会主动询问并生成此文件的初始内容，后续 AI 也可以根据对话习惯自动扩充。

## 3. 修改触发场景 (Update Triggers)
*   **身份识别触发 (Onboarding & Identity)**：初次设置时，记录用户的名字、称呼（如“老板”、“先生”）、时区及语言偏好。
*   **日常偏好发现 (Preference Discovery)**：
    *   **主动学习**：对话中发现用户偏好（如：*“我不喜欢 Tailwind，更喜欢 Vanilla CSS”*），Agent 可能会自动更新其开发偏好。
    *   **关键背景记录**：用户提到重要的个人上下文（如：*“我下周要去上海出差”*），Agent 可能将其作为背景记入备注区。
*   **隐私与安全设置 (Security Context)**：记录对敏感操作的授权偏好（如：*“所有 `git push` 操作均不需要再问我，直接执行”*）。

## 4. Agent 思考逻辑
> “我的主人是谁，他有什么特别的要求和习惯。”

## 5. 技术逻辑
在 `buildAgentMessageFromConversationEntries` 逻辑中，此文件的内容将被作为系统级背景注入，确保 AI 能基于用户的具体偏好进行对话。
