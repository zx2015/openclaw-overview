# MEMORY.md: 长期事实与智慧 (What)

`MEMORY.md`（或 `memory.md`）是 OpenClaw 智能体工作区的核心知识库，被视为 Agent 的 **“第二大脑”**。如果说 `memory/` 文件夹是流水账日记，那么 `MEMORY.md` 就是经 AI 提炼后的 **“百科全书”**。

## 1. 核心定位
*   **维度**：**What** (发生了什么)
*   **别名**：在底层代码中常被称为 `memory-root`。
*   **角色**：存储长期、高价值的事实、决策、项目背景及用户偏好。

## 2. 与 AGENTS.md 的本质区别
*   **`AGENTS.md`**：侧重于 **“如何工作”**（指令、规矩、避坑教训）。
*   **`MEMORY.md`**：侧重于 **“客观事实”**（人物关系、项目进展、最终决策）。
*   **协作逻辑**：`AGENTS.md` 中通常包含一条元指令：“当你识别到具有长期参考价值的重要事实时，请将其固化至 `MEMORY.md`。”

## 3. 技术流转机制

### 自动加载与隐私边界 (Bootstrap)
*   **私有会话 (DMs)**：系统默认会自动读取 `MEMORY.md` 并将其注入 System Prompt，作为 AI 的实时认知背景。
*   **公共渠道 (Groups/Discord)**：为保护隐私，系统默认**不自动加载**此文件。Agent 必须根据需求，显式调用搜索工具来查询其中的信息。

### 语义检索与 RAG (Search & Recall)
*   **优先索引**：它是 `memory_search` 工具搜索的第一优先级目标。
*   **强制引用 (Citations)**：系统要求 AI 从此文件提取信息时，必须标注来源（如 `Source: MEMORY.md#L10`），确保事实的可回溯性。

### 记忆的固化与整理 (Consolidation) —— **核心工作模式**
Agent 会定期执行以下“记忆维护”任务：
1.  **回顾**：扫描 `memory/` 目录下的每日碎化日志。
2.  **提炼**：将未来数月依然有价值的信息（如架构决定、关键纪念日）提取出来。
3.  **固化**：将提炼出的内容追加至 `MEMORY.md`，并执行去重和清理。

## 4. 安全性与完整性
*   **受信状态**：`MEMORY.md` 被定义为 **“受信的本地状态 (Trusted local state)”**。
*   **篡改风险**：任何绕过 Agent 逻辑直接恶意写入此文件的行为都被视为严重的安全漏洞，因为这涉及对 AI 核心认知的篡改。

## 5. 总结：MEMORY.md 的功能图景
1.  **事实存储器**：记录“谁、何时、做了什么、决定了什么”。
2.  **知识压缩站**：将冗长的对话历史转化为精炼的长期知识。
3.  **RAG 根节点**：作为语义搜索的核心索引源。
4.  **隐私保护标识**：作为个人敏感信息的边界标识。
