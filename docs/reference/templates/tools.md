# TOOLS.md: 本地工具备注与心得 (How with What)

`TOOLS.md` 是 OpenClaw 智能体对“工具”进行理解与适配的心得记录。

## 1. 核心定位
*   **维度**：**How with What** (我如何用好工具)
*   **角色**：Agent 对“工具使用”的本地化心得与环境配置备忘。
## 2. 具体用途
*   **非存在控制**：它不控制工具有没有，而是记录如何“用好”现有的工具（如特定的参数组合、已知的限制）。
*   **本地化心得**：存储环境特定信息，如本地摄像头的名称（如 "Logitech C920"）、特定的 SSH 连接别名、或是某个 Skills 在该环境下的特殊响应预期。
*   **环境特定备注 (Contextual Notes)**：AI 可以在此记录特定操作系统的路径偏好或特定的终端编码需求。
*   **用户指令记录**：作为用户留给 AI 的“小纸条”，指导 AI 在当前特定的环境中，应当如何适配用户的具体习惯。

## 3. 修改触发场景 (Update Triggers)
*   **环境适配触发 (Environment Adaptation)**：
    *   当 Agent 在用户设备上发现特定的硬件或服务信息时。*示例*：调用摄像头工具发现有多个摄像头，并确认主摄像头名称后，将其记入 `TOOLS.md`。
    *   记录特定路径或配置：例如某个自定义脚本的绝对路径，或特定服务器的 SSH 连接端口。
*   **避坑指南触发 (Specific Skill Notes)**：
    *   当 Agent 使用某个 Skill（如 `whatsapp` 或 `browser`）遇到特定限制或报错，并找到了规避方法时，会更新此文件以避免下次犯错。
*   **用户配置说明触发 (User-provided Parameters)**：
    *   用户直接指令：*“记住，我的服务器 IP 是 192.168.1.100，以后用这个连接。”* Agent 会将此类信息作为工具参数备注存入。

## 4. Agent 思考逻辑
> “在这个环境下，这个工具得这么用才对。”
